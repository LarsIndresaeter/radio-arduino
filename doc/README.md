# Documentation

Documentation is inspired by [divio](https://docs.divio.com/documentation-system/).

> **A word of notice:**
> I have written the code in this repository with the [motivation](./explanation/motivation-for-this-work.md) to learn, not to make a product.

## [Tutorials ](./tutorials/README.md)

Lessons that take the reader by the hand through a series of steps to complete a project. For example:

* how to compile
* how to download software to arduino
* how to publish data to a mqtt broker

## [How-to guides](./how-to-guides/README.md)

Guides that take the reader through the steps to solve a real-world problem. For example:

* how to add a new command
* how to debug

## [Reference guides](./reference-guides/README.md)

Technical descriptions of the machinery and how to operate it. For example:

* which commands are available
* which command line tools are available

## [Explanation](./explanation/README.md)

Explanation, or discussions, clarify and illuminate a particular topic. For example:

* how does the protocol work
* how does the tools-kit works
