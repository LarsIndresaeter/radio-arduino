. Documentation

Inspired by https://docs.divio.com/documentation-system/how-to-guides/

Tutorials
=========

Lessons that take the reader by the hand through a series of steps to complete a project.

For example:
* how to compile
* how to download software to arduino
* how to publish data to a mqtt broker

How-to guides
=============

Guides that take the reader through the steps to solve a real-world problem. 

For example:
* how to add a new command

Reference guides
================

Technical descriptions of the machinery and how to operate it.

For example:
* How conan and cmake is used to configure the build of the C++ code

Explanations
============

Explanation, or discussions, clarify and illuminate a particular topic.

For example:
* Why this project is organized as a mono-repo and what the tradeoffs are if you decide to split it into several repositories.
